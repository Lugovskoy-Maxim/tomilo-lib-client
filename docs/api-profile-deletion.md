# API удаления профиля (бэкенд)

Фронтенд вызывает следующие эндпоинты. Их нужно реализовать на бэкенде в той же папке, где находится клиент.

## Модель пользователя

Добавьте в схему пользователя два поля (ISO 8601 даты):

- `scheduledDeletionAt` (Date, optional) — дата и время, после которых профиль считается удалённым (обычно now + 7 дней).
- `deletedAt` (Date, optional) — дата фактического «удаления» (проставляется кроном/задачей через 7 дней после `scheduledDeletionAt`; профиль остаётся в БД, но помечен как удалённый).

В ответах `GET /users/profile` и `GET /users/:id` возвращайте эти поля, чтобы фронт показывал баннеры и метку «Удалён».

## Эндпоинты

### POST /users/profile/schedule-deletion

- **Авторизация:** обязательна (JWT / сессия текущего пользователя).
- **Тело:** пустое или `{}`.
- **Логика:**
  - Установить у текущего пользователя `scheduledDeletionAt = now + 7 days` (по серверному времени).
  - Если уже есть `scheduledDeletionAt` в будущем — можно оставить как есть или обновить на +7 дней от текущего момента (на ваше усмотрение).
- **Ответ:** 200 и обновлённый объект пользователя (в том же формате, что и `GET /users/profile`), включая `scheduledDeletionAt`.

### POST /users/profile/cancel-deletion

- **Авторизация:** обязательна.
- **Тело:** пустое или `{}`.
- **Логика:**
  - Обнулить у текущего пользователя `scheduledDeletionAt` (например, `null` или удалить поле).
- **Ответ:** 200 и обновлённый объект пользователя без `scheduledDeletionAt` (или с `scheduledDeletionAt: null`).

## Фоновая задача (крон/воркер)

Раз в день (или чаще) выполняйте задачу:

- Найти пользователей, у которых `scheduledDeletionAt <= now` и `deletedAt` не задано.
- Для каждого установить `deletedAt = now` (и при желании очистить `scheduledDeletionAt`).

После этого профиль остаётся в БД, но считается удалённым: логин по этому аккаунту можно запретить, а на фронте отображается метка «Удалён» и текст о том, что данные сохранены, но не используются. Полное удаление данных — по запросу на support@tomilo-lib.ru (как указано в уведомлении при удалении).
